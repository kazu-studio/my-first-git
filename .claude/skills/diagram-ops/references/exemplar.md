# 模範解答パターン・品質チェックリスト

## 成功する図解の構造

```
1. ヘッダー（ダークネイビーグラデーション）
   └─ 手順名 + 全ステップ数 + 対象ツール一覧

2. スティッキー進捗バー
   └─ 「進捗 ██░░░░ 0 / N」（JS更新）

3. すごろく型フロー（上から順番に）
   ├─ 通常ステップ（青）
   │   └─ 番号バッジ / アクション / ツール / 確認事項
   ├─ 注意ステップ（橙）
   │   └─ 上記 + 注意文 + 分岐カード（あれば）
   ├─ 危険ステップ（赤）
   │   └─ 上記 + 「要注意」バッジ + 赤警告ボックス
   └─ 各ステップに「今ここ」ボタン（JS）

4. 完了カード（緑）
   └─ 完了後の確認・報告先
```

---

## 品質チェックリスト

作成後、以下を全て確認する。

### コンテンツ

- [ ] 手順の全ステップ数がヘッダーに明記されている
- [ ] 各ステップに「使用ツール」が具体的に書かれている
- [ ] 各ステップに「確認事項」が「〜であることを確認」形式で書かれている
- [ ] 注意・危険ステップが色（橙・赤）で区別されている
- [ ] 分岐点に「OK → Step X」「エラー → 〜」のカードがある
- [ ] 危険ステップに赤い警告ボックスがある
- [ ] 完了カードがある

### 機能

- [ ] 「今ここ」ボタンが全ステップにある
- [ ] 「今ここ」を押すと進捗バーが更新される
- [ ] ページ再読み込み後もlocalStorageで現在位置が復元される

### デザイン

- [ ] Lucide iconを使用（絵文字禁止）
- [ ] ヘッダーはダークネイビーグラデーション
- [ ] ステップカードの色分けが視覚的に明確（青/橙/赤）
- [ ] スマホでも読みやすいレスポンシブ対応
- [ ] ステップ間に矢印（`arrow-down`）がある

## よくある失敗パターンと修正方法

| 失敗 | 原因 | 修正 |
|------|------|------|
| ツール名が曖昧 | 「システムを開く」など | 「XX管理画面 > YYメニュー」に具体化 |
| 全体像が掴めない | ステップ数がヘッダーにない | ヘッダーに「全N ステップ」を追記 |
| 危険かどうかわからない | 全ステップが同じ色 | 注意レベルの色分けを適用 |
| 分岐先が不明 | 「OKなら次へ」しか書いていない | 「エラーの場合 → 〜」のカードも追加 |
| 今どこにいるか不明 | 進捗バーが機能していない | JS の `TOTAL_STEPS` 変数を正しいN に更新 |

## ステップ数別の推奨レイアウト

| ステップ数 | 推奨 |
|-----------|------|
| 〜5ステップ | 縦並びのみ（シンプル） |
| 6〜10ステップ | 縦並び + 各ステップにツール・確認・注意を詳細表示 |
| 11〜20ステップ | 大手順をセクション（Phase）でグループ化 |
| 21ステップ〜 | Phaseで分割し、手順書を複数ページに分ける検討 |

## Phase（大手順のグループ化）

ステップが多い場合はPhaseでまとめる。

```html
<!-- Phaseヘッダー -->
<div class="flex items-center gap-3 my-6">
  <div class="w-10 h-10 bg-slate-700 text-white rounded-xl flex items-center justify-center font-bold">
    1
  </div>
  <div>
    <h2 class="text-lg font-bold text-slate-700">Phase 1: 事前確認</h2>
    <p class="text-sm text-slate-500">Step 1〜3</p>
  </div>
  <div class="flex-1 h-px bg-slate-300"></div>
</div>
```

## 成功する図解が満たす条件

新人が図解を見た後、以下の3つを自分で答えられる状態になっていること:

1. **全体像**: 「この手順は全部で何ステップある」
2. **現在地**: 「今自分はStep Xにいる」（「今ここ」ボタンで確認）
3. **次のアクション**: 「次はXX画面で〇〇を確認してからYYを実行する」
